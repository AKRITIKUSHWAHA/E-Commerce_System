## ✨ Features

### 🧑‍💼 Admin Panel
- Full product management — Add, Edit, Delete with multi-image upload
- Stock management — auto-deduction on order, low stock alerts
- Order management — update status (Shipped → Delivered → Cancelled)
- Category management — create & toggle visibility on storefront
- Customer management — view all users, ban/unban accounts
- Analytics dashboard — revenue chart (6 months), page views, today's visits
- Auto-refresh dashboard every 30 seconds

### 🛒 Customer Store
- Browse products with filters — category, price range, rating, in-stock
- Product image slider with arrows and dot indicators
- Wishlist & Cart with quantity control
- Multi-image product gallery with thumbnails
- Stock count display on each product card
- Search products from navbar

### 💳 Checkout & Payments
- Delivery address with landmark & alternate phone number
- Quantity selector before checkout
- Razorpay integration — UPI, Cards, Net Banking, Wallets
- HMAC-SHA256 payment signature verification
- Order confirmation page with full details after payment

### 📦 Order Tracking
- Real-time order status: Confirmed → Shipped → Delivered
- Visual step-by-step tracking bar
- Order history with item images & delivery address
- Click product image in order to go back to product page

### ⭐ Reviews & Ratings
- Star rating system (1–5 stars)
- Review option unlocked only after order is delivered
- Average rating displayed on product cards and detail page

### 🔐 Authentication
- JWT-based login & registration
- Bcrypt password hashing
- Role-based access control (Admin / Customer)
- Change password from profile section
- Protected routes for admin & user pages

## 🚀 Getting Started

### Prerequisites

Make sure you have the following installed:
- [Node.js](https://nodejs.org/) v16+
- [MySQL](https://www.mysql.com/) v8+
- npm

---

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/steepray-ecommerce.git
cd steepray-ecommerce
```

---

### 2. Backend Setup

```bash
cd Backend
npm install
```

Create `.env` file (see [Environment Variables](#-environment-variables) section below)

Start the backend server:
```bash
node server.js
```

Backend runs on: `http://localhost:5000`

---

### 3. Frontend Setup

```bash
cd Frontend
npm install
npm run dev
```

Frontend runs on: `http://localhost:5173`

---

## 🔐 Environment Variables

Create a `.env` file inside the `Backend/` folder:

```env
# Server
PORT=5000

# Database
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=ecommerce_db

# JWT
JWT_SECRET=your_jwt_secret_key

# Razorpay
RAZORPAY_KEY_ID=rzp_test_xxxxxxxxxxxxxxxx
RAZORPAY_KEY_SECRET=your_razorpay_secret
```

> ⚠️ **Important:** Do NOT add comments after values with `=` signs — it breaks the key parsing.

## 🗄️ Database Setup

### 1. Create Database

```sql
CREATE DATABASE ecommerce_db;
USE ecommerce_db;
```

### 2. Tables

```sql
-- Users
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  role ENUM('customer', 'admin') DEFAULT 'customer',
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Categories
CREATE TABLE categories (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  slug VARCHAR(100) UNIQUE NOT NULL,
  emoji VARCHAR(10),
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Products
CREATE TABLE products (
  id INT AUTO_INCREMENT PRIMARY KEY,
  seller_id INT,
  category_id INT,
  name VARCHAR(200) NOT NULL,
  description TEXT,
  brand VARCHAR(100),
  price DECIMAL(10,2) NOT NULL,
  original_price DECIMAL(10,2) NOT NULL,
  stock INT DEFAULT 0,
  images JSON,
  sizes JSON,
  colors JSON,
  tags JSON,
  avg_rating DECIMAL(3,1) DEFAULT 0,
  rating_count INT DEFAULT 0,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (category_id) REFERENCES categories(id)
);

-- Orders
CREATE TABLE orders (
  id INT AUTO_INCREMENT PRIMARY KEY,
  customer_id INT,
  customer_name VARCHAR(100),
  customer_email VARCHAR(100),
  customer_phone VARCHAR(15),
  address TEXT,
  total_amount DECIMAL(10,2),
  payment_method VARCHAR(50),
  payment_status ENUM('pending','paid','failed','refunded') DEFAULT 'pending',
  status ENUM('pending','confirmed','shipped','delivered','cancelled') DEFAULT 'pending',
  notes TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (customer_id) REFERENCES users(id)
);

-- Order Items
CREATE TABLE order_items (
  id INT AUTO_INCREMENT PRIMARY KEY,
  order_id INT,
  product_id INT,
  seller_id INT,
  product_name VARCHAR(200),
  price DECIMAL(10,2),
  quantity INT,
  size VARCHAR(50),
  color VARCHAR(50),
  FOREIGN KEY (order_id) REFERENCES orders(id),
  FOREIGN KEY (product_id) REFERENCES products(id)
);

-- Reviews
CREATE TABLE reviews (
  id INT AUTO_INCREMENT PRIMARY KEY,
  product_id INT,
  user_id INT,
  user_name VARCHAR(100),
  rating TINYINT NOT NULL CHECK (rating BETWEEN 1 AND 5),
  comment TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (product_id) REFERENCES products(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- Page Views
CREATE TABLE page_views (
  id INT AUTO_INCREMENT PRIMARY KEY,
  page VARCHAR(255),
  ip VARCHAR(45),
  user_id INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 📡 API Endpoints

### Auth
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/auth/register` | Register new user |
| POST | `/api/auth/login` | Login & get JWT token |
| GET | `/api/auth/me` | Get current user info |
| PUT | `/api/auth/update-password` | Change password |

### Products (Public)
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/products` | Get all products (with filters) |
| GET | `/api/products/:id` | Get single product |
| GET | `/api/categories` | Get all active categories |
| GET | `/api/products/:id/reviews` | Get product reviews |
| POST | `/api/products/:id/reviews` | Add review (auth required) |

### Orders (Auth Required)
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/orders/create-razorpay-order` | Create Razorpay order |
| POST | `/api/orders/verify-payment` | Verify & place order |
| GET | `/api/user/orders` | Get current user's orders |

### Admin (Admin Only)
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/admin/dashboard` | Get dashboard stats |
| GET | `/api/admin/orders` | Get all orders |
| PUT | `/api/admin/orders/:id/status` | Update order status |
| GET | `/api/admin/users` | Get all users |
| GET | `/api/admin/categories` | Get all categories |
| PUT | `/api/admin/categories/:id/toggle` | Toggle category |
| PATCH | `/api/admin/products/:id/toggle-stock` | Toggle stock |
| GET | `/api/admin/page-views` | Get page view analytics |
| POST | `/api/track/view` | Track page visit |

### Seller / Product Management
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/seller/products` | Get seller's products |
| POST | `/api/seller/products` | Create new product |
| PUT | `/api/seller/products/:id` | Update product |
| DELETE | `/api/seller/products/:id` | Delete product |

## 💡 Key Implementation Notes

- **Razorpay** — Payment flow: Create Order → Open Popup → HMAC Verify → Save Order
- **Stock** — Auto decrements on successful payment; admin adds stock additively (not replace)
- **Reviews** — Only users with a `delivered` order for that product can submit a review
- **Images** — Stored in `Backend/uploads/` via Multer; served as static files
- **Auto Refresh** — Home, Admin Dashboard & User Account auto-refresh every 30 seconds silently
- **Cart** — Cleared after successful checkout; items locked in state before clear to show success page
